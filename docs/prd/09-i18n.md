# Part 9 — i18n (Localisation)

## Overview

minus-tracker is an Italian-market product. Italian (`it`) is the primary locale; English (`en`)
is secondary. The active language is configurable and persists across invocations via a settings
file. All user-facing CLI text is localised. The programmatic library API (error `.message`
strings, `GainsReport.warnings[]`) stays in English for stability.

## Supported Locales

| Code | Language | Default? |
| ---- | -------- | -------- |
| `it` | Italiano | Yes      |
| `en` | English  | No       |

## Locale Resolution (highest to lowest priority)

1. `--lang <it|en>` — CLI flag (one-shot, not persisted)
2. `MINUS_TRACKER_LANG=it` — environment variable
3. `~/.config/minus-tracker/config.json` → `"locale"` field
4. Hardcoded default: `"it"`

An invalid locale value (any value outside `it`|`en`) is a hard error at CLI startup: exit code 2,
message: `"Unsupported locale \"<value>\". Use: it, en"` (always in English, since we cannot know
which locale to render the error in).

## Settings File

### Path

| Platform      | Path                                                                                               |
| ------------- | -------------------------------------------------------------------------------------------------- |
| macOS / Linux | `$XDG_CONFIG_HOME/minus-tracker/config.json` (falls back to `~/.config/minus-tracker/config.json`) |
| Windows       | `%APPDATA%\minus-tracker\config.json`                                                              |

Path resolution uses `node:os`, `node:path`, `node:fs` only — zero new runtime dependencies.

### Format

```json
{
  "locale": "it"
}
```

The file holds only explicitly-set values. Absent fields fall through to defaults, making the file
forward-compatible when new config keys are added.

### Full settings interface

```typescript
interface MinusTrackerConfig {
  locale?: SupportedLocale;
  // reserved: defaultMethod?: LotMethod;
}
```

## Architecture

### Module layout

```
src/
├── i18n/
│   ├── types.ts       ← LocaleStrings interface, SupportedLocale type, MinusTrackerConfig
│   ├── it.ts          ← Italian locale object (implements LocaleStrings)
│   ├── en.ts          ← English locale object (implements LocaleStrings)
│   ├── settings.ts    ← resolveLocale(), saveLocale(), readConfig(), path helpers
│   └── index.ts       ← getStrings(), re-exports
├── parser/
│   ├── index.ts       ← DEGIROParser (uses WarningEntry internally)
│   └── warnings.ts    ← WarningEntry discriminated union + warningToEnglish()
├── errors.ts          ← ParseError (structured), CalculationError (structured)
├── calculator/
│   └── index.ts       ← Calculator class
├── types.ts           ← Transaction, MatchedLot, GainsReport, LotMethod
└── index.ts           ← Public library exports (no i18n re-exported here)

src/cli/
├── index.ts           ← CLI entry; resolves locale once, passes `s: LocaleStrings` to commands
├── renderer.ts        ← Table/summary rendering
└── commands/
    ├── calc.ts
    ├── validate.ts
    ├── rates.ts
    └── config.ts      ← NEW: `minus-tracker config` command
```

### Tree-shakeability guarantee

`src/index.ts` (library entry) **never** imports from `src/i18n/`. The `src/i18n/` subgraph is
only reachable from `src/cli/index.ts`. Bundlers targeting `src/index.ts` include zero locale
code.

### Locale wiring in the CLI

```typescript
// src/cli/index.ts — pseudocode
const cliLang = args["lang"] as string | undefined;
const locale = resolveLocale(cliLang); // throws + exit 2 on invalid value
const s = getStrings(locale);

// `s` is passed as a parameter to every command function.
// Command functions never call getStrings() themselves.
```

Both locale objects are statically imported in `src/i18n/index.ts` — no dynamic imports. Since
the i18n module is CLI-only, bundle size is not a concern.

## `LocaleStrings` Interface

File: `src/i18n/types.ts`

```typescript
export type SupportedLocale = "it" | "en";

export interface LocaleStrings {
  // ── Number formatting ─────────────────────────────────────────────────────
  /** BCP 47 tag used with Intl.NumberFormat for EUR amounts in table output. */
  numberLocale: "it-IT" | "en-US";

  // ── Hard error messages ───────────────────────────────────────────────────
  errorInvalidCsv: string;
  errorMissingColumn: (columnName: string) => string;
  errorNoOpenLots: (isin: string, date: string) => string;

  // ── Soft warnings (per-row) ───────────────────────────────────────────────
  warnMissingIsin: (row: number) => string;
  warnUnsupportedCurrency: (row: number, currency: string) => string;
  warnNoEcbRate: (row: number, currency: string, date: string) => string;
  warnQuantityZero: (row: number) => string;

  // ── calc command: multi-year warning ─────────────────────────────────────
  warnMultipleYears: string;

  // ── calc command: table column headers ───────────────────────────────────
  headerMethod: string;
  headerTaxYear: string;
  headerIsin: string;
  headerProduct: string;
  headerQty: string;
  headerBuyDate: string;
  headerSellDate: string;
  headerBuyEur: string;
  headerSellEur: string;
  headerGainLoss: string;

  // ── calc command: summary block ───────────────────────────────────────────
  summaryPlusvalenze: string;
  summaryMinusvalenze: string;
  summaryNetResult: string;
  summaryWarnings: string;
  summaryGenerated: string;

  // ── validate command ──────────────────────────────────────────────────────
  validateOk: (count: number, hardErrors: number) => string;
  validateWarn: (count: number, reason: string) => string;

  // ── rates command ─────────────────────────────────────────────────────────
  ratesCoverage: (start: string, end: string, currencies: string) => string;
  ratesGapsNone: string;
  ratesGaps: (list: string) => string;
  ratesUpdateFetching: string;
  ratesUpdateDone: (addedCount: number) => string;
  ratesSnapshotWritten: (filePath: string) => string;

  // ── config command ────────────────────────────────────────────────────────
  configLangSet: (lang: SupportedLocale) => string;
  configCurrentLang: (lang: SupportedLocale) => string;

  // ── Legal disclaimer — always Italian, regardless of active locale ────────
  /** Rendered at the foot of every `calc` table output. Never translated. */
  disclaimer: string;
}
```

The compiler enforces exhaustiveness: adding a key to `LocaleStrings` causes a type error in
both `it.ts` and `en.ts` until both locale objects are updated.

## Structured Errors

The library's `ParseError` and `CalculationError` carry machine-readable fields alongside their
English `.message`. The CLI reads these fields to render locale-aware output.

### `src/errors.ts`

```typescript
export class ParseError extends Error {
  readonly code: "INVALID_CSV" | "MISSING_COLUMN";
  readonly columnName?: string;

  constructor(code: "INVALID_CSV");
  constructor(code: "MISSING_COLUMN", columnName: string);
  constructor(code: ParseError["code"], columnName?: string) {
    const msg =
      code === "INVALID_CSV"
        ? "Invalid CSV: unable to parse"
        : `Missing required column: ${columnName}`;
    super(msg);
    this.name = "ParseError";
    this.code = code;
    this.columnName = columnName;
  }
}

export class CalculationError extends Error {
  readonly isin: string;
  readonly date: string;

  constructor(isin: string, date: string) {
    super(`No open lots for ISIN ${isin} on ${date}`);
    this.name = "CalculationError";
    this.isin = isin;
    this.date = date;
  }
}
```

### CLI rendering pattern

```typescript
// In CLI catch blocks — pseudocode
if (err instanceof ParseError) {
  const msg =
    err.code === "INVALID_CSV"
      ? s.errorInvalidCsv
      : s.errorMissingColumn(err.columnName!);
  process.stderr.write(msg + "\n");
}
if (err instanceof CalculationError) {
  process.stderr.write(s.errorNoOpenLots(err.isin, err.date) + "\n");
}
```

## Structured Warnings

### `src/parser/warnings.ts`

```typescript
export type WarningEntry =
  | { code: "MISSING_ISIN"; row: number }
  | { code: "UNSUPPORTED_CURRENCY"; row: number; currency: string }
  | { code: "NO_ECB_RATE"; row: number; currency: string; date: string }
  | { code: "QUANTITY_ZERO"; row: number };

export function warningToEnglish(w: WarningEntry): string {
  switch (w.code) {
    case "MISSING_ISIN":
      return `Row ${w.row}: missing ISIN — skipped`;
    case "UNSUPPORTED_CURRENCY":
      return `Row ${w.row}: unsupported currency ${w.currency} — skipped`;
    case "NO_ECB_RATE":
      return `Row ${w.row}: no ECB rate for ${w.currency} on ${w.date} — skipped`;
    case "QUANTITY_ZERO":
      return `Row ${w.row}: quantity is 0 — skipped`;
  }
}
```

`DEGIROParser` populates an internal `_warningEntries: WarningEntry[]`. The public
`GainsReport.warnings: string[]` getter returns `_warningEntries.map(warningToEnglish)` —
satisfying the Part 2 contract. The CLI accesses the structured entries via an internal
`parser.warningEntries` getter (not re-exported from `src/index.ts`) to render locale-aware
messages.

## Number Formatting

EUR amounts in the `calc` table output are formatted with `Intl.NumberFormat` using the locale's
`numberLocale` tag:

```typescript
const fmt = new Intl.NumberFormat(s.numberLocale, {
  minimumFractionDigits: 2,
  maximumFractionDigits: 2,
});
// Italian: "1.234,00"   English: "1,234.00"
```

Internal computation always uses JavaScript `number` (IEEE 754). Formatting is presentation-only
and never fed back into calculations.

## `config` Command Spec

```
minus-tracker config --lang <it|en>   Persist locale to settings file
minus-tracker config --show           Print current effective config
```

**Example — set language to English:**

```
$ minus-tracker config --lang en
Language set to: en
```

**Example — show config (Italian locale active):**

```
$ minus-tracker config --show
Lingua corrente: it
```

Exit codes: 0 success, 2 invalid `--lang` value.

The `--lang` global flag (available on all commands) overrides locale for a single invocation
without writing to the settings file:

```
minus-tracker calc --lang en trades.csv
```
